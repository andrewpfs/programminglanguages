Consider the following pseudo code.

compute(n) = n
respond(n) = 0

main = print(respond(compute(7)))

1. If this pseudocode were implemented in Java, C#, Python, C++, etc., discuss 
what will happen when main is executed (not just the output, but the behavior 
of code) and provide reasons for that behavior.

If this were to be implemented in the given languages, compute(7) would be executed first and return 7. It would then compute respond(7) and return 0. The code would then print 0. This is because these languages have Eager/Strict evaluation, which evaluates expressions immediately.

2. If this pseudocode were implemented in a language like Haskell, discuss 
what will happen when main is executed (not just the output, but the behavior 
of code) and provide reasons for that behavior.

If this were to be implemented in Haskell, it would skip compute(7) for now because it isn’t needed. It would then compute respond, ignoring the argument and return 0 since respond() will always return 0 regardless of the input. It would then print 0. Unlike the other languages, Haskell has Lazy Evaluation which defers the computation of expressions until it is needed. In this case, compute(7) will likely never be evaluated because respond(n) = 0 doesn’t depend on n.
